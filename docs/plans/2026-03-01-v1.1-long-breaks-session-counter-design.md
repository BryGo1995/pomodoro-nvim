# pomodoro.nvim v1.1 Design ‚Äî Long Breaks & Session Counter

**Date:** 2026-03-01
**Status:** Approved

---

## Overview

Add proper Pomodoro method session tracking to the plugin:
- Long breaks triggered after every N work sessions (configurable, default 4)
- Dot indicators showing progress within the current set
- Daily pomodoro count persisted to disk, resetting at midnight
- Statusline format: `üçÖ√ó7 ‚óè‚óè‚óè‚óã 24:30`

---

## New Config Keys

```lua
return {
  work_minutes        = 25,
  break_minutes       = 5,
  long_break_minutes  = 15,   -- default: 15
  long_break_interval = 4,    -- long break after every N work sessions, default: 4
}
```

All four keys remain optional ‚Äî missing keys fall back to defaults silently.

---

## State Machine

Replace `state.is_break` (bool) with `state.phase` (string).

**Valid phases:** `"idle"` | `"work"` | `"break"` | `"long_break"`

**New state fields:**
```lua
local state = {
  phase             = "idle",
  running           = false,
  remaining_seconds = 0,
  timer_handle      = nil,
  _generation       = 0,
  set_count         = 0,    -- pomodoros completed in current set (resets after long break)
  daily_count       = 0,    -- total pomodoros today (persists to disk)
  config            = vim.deepcopy(defaults),
}
```

**Transition logic when work hits zero:**
1. Increment `set_count` and `daily_count`
2. Save `daily_count` to disk
3. If `set_count >= config.long_break_interval` ‚Üí start `"long_break"`, reset `set_count = 0`
4. Otherwise ‚Üí start `"break"`

**Transition logic when break/long_break hits zero:**
- Set `running = false`, `phase = "idle"`
- Fire notification: "Break over! Ready to focus?" / "Long break over! Ready to focus?"

**`skip()` logic:**
- `"work"` ‚Üí same break-or-long-break decision as `_tick` (but no counter increment)
- `"break"` or `"long_break"` ‚Üí `running = false`, `phase = "idle"`
- `"idle"` ‚Üí no-op (existing guard)

## Statusline Format

| Phase | Format | Notes |
|---|---|---|
| `"work"` | `üçÖ√óN ‚óè‚óè‚óè‚óã MM:SS` | N = daily_count, dots = set progress |
| `"break"` | `‚òï√óN ‚óè‚óè‚óè‚óã MM:SS` | dots show set progress toward long break |
| `"long_break"` | `üåô√óN MM:SS` | no dots ‚Äî set just reset |
| `"idle"` | `""` | empty string |

**Dot generation:** `long_break_interval` total dots ‚Äî filled `‚óè` for each completed session in `set_count`, empty `‚óã` for remaining.
Example: `set_count=2`, `interval=4` ‚Üí `‚óè‚óè‚óã‚óã`

## Persistence

**File:** `vim.fn.stdpath("data") .. "/pomodoro-daily.json"`
**Format:** `{"date":"2026-03-01","count":7}`

**On `setup()`:**
- Read file with `vim.fn.readfile()`
- Parse JSON with `vim.fn.json_decode()`
- If stored date == today ‚Üí restore `daily_count`
- If stored date != today (or file missing/malformed) ‚Üí `daily_count = 0`

**On each completed work session:**
- Write updated count with `vim.fn.writefile(vim.fn.json_encode({date=today, count=daily_count}), path)`

No external dependencies ‚Äî `vim.fn.json_encode` / `vim.fn.json_decode` are built into NeoVim.

## Changes to Existing Functions

| Function | Change |
|---|---|
| `_start_phase(phase)` | Takes phase string instead of `is_break` bool; looks up duration by phase |
| `_tick(gen)` | New counter/persistence logic on work completion; updated phase transitions |
| `skip()` | Updated to use phase string; applies counter decision for work‚Üíbreak |
| `statusline()` | Builds new format string from phase, daily_count, set_count, remaining_seconds |
| `_reset_state()` | Resets phase, set_count, daily_count (daily_count = 0 in tests) |
| `_set_state()` | valid_keys updated: add phase, set_count, daily_count; remove is_break |
| `setup()` | Calls persistence load on startup |
| `defaults` | Add long_break_minutes = 15, long_break_interval = 4 |

---

## Out of Scope

- Resetting the session counter manually (`:PomodoroReset`)
- Showing yesterday's count
- Any UI beyond the statusline string
